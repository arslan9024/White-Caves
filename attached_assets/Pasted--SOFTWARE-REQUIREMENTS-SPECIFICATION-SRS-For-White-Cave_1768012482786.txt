## **1. INTRODUCTION**

### **1.1 Purpose**
This document defines the complete functional and non-functional requirements for the White Caves Real Estate AI Platform. It serves as the authoritative specification for developers, testers, project managers, and stakeholders to ensure all implemented features are properly connected, surfaced in the UI, and deliver measurable business value.

### **1.2 Scope**
The platform is a comprehensive AI-powered real estate management system serving the Dubai luxury property market. It integrates property listing management, intelligent customer relationship management (CRM), automated AI assistants, financial tracking, and multi-role dashboards into a unified web application.

**In-Scope:**
- AI Assistant dashboards (Zoe, Aurora, Linda, Clara, etc.)
- Multi-role user interfaces and navigation
- Property and lead management workflows
- WhatsApp Business API integration
- Real-time data visualization and analytics
- UAE-specific compliance features (RERA, DLD, Ejari)

**Out-of-Scope:**
- Mobile native applications (iOS/Android)
- Blockchain-based property tokenization
- Advanced predictive market analytics using external data feeds
- Physical hardware integration (smart locks, sensors)

### **1.3 Definitions, Acronyms, and Abbreviations**
| Term | Definition |
|------|------------|
| **AI Assistant** | A specialized software agent managing a specific business domain (e.g., Zoe for executive functions) |
| **CRM** | Customer Relationship Management system |
| **RERA** | Real Estate Regulatory Agency (Dubai) |

| **AED** | UAE Dirham (currency) |

## **2. OVERALL DESCRIPTION**

### **2.1 Product Perspective**
The White Caves platform is a standalone web application that will integrate with several external systems:
- **WhatsApp Business API** for customer communications
- **Google Workspace** for calendar and document management
- **Stripe Payment Gateway** for transaction processing
- **MongoDB Atlas** for cloud database hosting
- **Firebase Authentication** for user identity management

### **2.2 Product Functions**
| Function Category | Key Capabilities |
|-------------------|------------------|
| **Property Management** | List, search, filter, compare properties with UAE-specific fields |
| **Lead Management** | Capture, score, assign, and track leads through sales pipeline |
| **AI Automation** | 14+ specialized AI assistants for different business functions |
| **Communication** | Integrated WhatsApp, email, and in-app messaging |
| **Document Management** | Tenancy contracts, compliance documents, ownership papers |
| **Analytics & Reporting** | Real-time dashboards, performance metrics, financial reports |
| **User Management** | Role-based access control with 5 distinct user types |

### **2.3 User Characteristics**
| User Role | Technical Proficiency | Frequency of Use | Primary Goals |
|-----------|----------------------|------------------|--------------|
| **Company Owner** | Medium-High | Daily | Monitor business performance, manage team, make strategic decisions |
| **Real Estate Agent** | Medium | Daily | Manage assigned leads, schedule viewings, close deals |
| **Property Buyer** | Low-Medium | Weekly | Search properties, schedule viewings, submit inquiries |
| **Property Seller/Landlord** | Medium | Weekly | List properties, track interest, manage documents |
| **Tenant** | Low | Monthly | Apply for rentals, sign contracts, submit maintenance requests |
| **System Administrator** | High | Daily | Manage users, configure system, monitor performance |

### **2.4 Constraints**
1. **Regulatory Compliance**: Must adhere to UAE RERA and DLD regulations
2. **Language Support**: Must support both English and Arabic (RTL layout)
3. **Performance**: Dashboard load time < 3 seconds on 4G connection
4. **Security**: All customer data must be encrypted at rest and in transit
5. **Browser Support**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+

### **2.5 Assumptions and Dependencies**
1. **Assumption**: Company owner (arslanmalikgoraha@gmail.com) has exclusive access to full CRM
2. **Assumption**: WhatsApp Business API approval will be obtained before launch
3. **Dependency**: MongoDB Atlas cluster will be provisioned before deployment
4. **Dependency**: Google Workspace organization account exists for integration
5. **Assumption**: All agents have smartphones with WhatsApp installed

## **3. SPECIFIC REQUIREMENTS**

### **3.1 Functional Requirements**

#### **3.1.1 User Interface Requirements**

**UI-FR-001: Unified Navigation System**
The system shall provide a consistent navigation structure across all pages consisting of:
1. A fixed top navigation bar with: Logo, Global Search, Theme Toggle, Notifications, User Profile
2. A collapsible sidebar (280px expanded / 72px collapsed) with hierarchical menu
3. Dashboard-specific tab navigation (maximum 6 tabs per dashboard)
*Priority: P0 (Critical)*

**UI-FR-002: Role-Based Dashboard Access**
The system shall display different dashboard sets based on user role:
1. Company Owner: Access to ALL 14+ AI assistant dashboards
2. Real Estate Agent: Access to Agent Dashboard, Leads CRM, and Property Search
3. Property Buyer/Seller: Access to respective portals only
4. System shall prevent unauthorized dashboard access with clear "Access Denied" message
*Priority: P0 (Critical)*

**UI-FR-003: Real-Time Data Display**
All UI components displaying data shall:
1. Show data source and last update timestamp
2. Provide manual refresh control
3. Display loading states using skeleton screens
4. Show empty states with actionable prompts when no data exists
5. Display error states with recovery options when data fetch fails
*Priority: P1 (High)*

**UI-FR-004: Action Feedback System**
All user interactions shall provide immediate visual feedback:
1. Buttons: Hover → Active → Loading → Success/Error states
2. Forms: Real-time validation → Submission → Result confirmation
3. Data changes: Visual indicators showing what changed
4. Notifications: Toast messages for system events
*Priority: P1 (High)*

#### **3.1.2 CRM Core Requirements**

**CRM-FR-001: Exclusive Company Owner Access**
The full CRM functionality shall be accessible ONLY to the user with email `arslanmalikgoraha@gmail.com`.
1. All other users shall see "Access Restricted - Company Owners Only" message
2. Attempted access shall be logged for security auditing
3. Owner access shall bypass all other role-based restrictions
*Priority: P0 (Critical)*

**CRM-FR-002: Lead Management Pipeline**
The system shall manage leads through a 7-stage pipeline:
1. **New**: Initial inquiry received
2. **Contacted**: Agent has made first contact
3. **Qualified**: Lead meets minimum criteria (score > 60)
4. **Proposal**: Formal proposal sent to client
5. **Negotiation**: Price/terms under discussion
6. **Won**: Deal closed successfully
7. **Lost**: Deal lost to competition or withdrawn
*Priority: P1 (High)*

**CRM-FR-003: Lead Scoring Automation**
The system shall automatically score leads based on:
1. **Budget Clarity** (0-25 points): Has the client specified a budget range?
2. **Property Specificity** (0-25 points): Has the client identified specific properties?
3. **Timeline Urgency** (0-25 points): Does the client have a specific move-in date?
4. **Communication Responsiveness** (0-25 points): How quickly does the client respond?
*Priority: P1 (High)*

**CRM-FR-004: Document Management**
The system shall manage UAE-specific real estate documents:
1. Automatically generate Ejari-compliant tenancy contracts
2. Track RERA permit numbers and expiration dates
3. Store and categorize: Passport copies, Visa pages, Emirates ID, Title Deeds
4. Provide document expiry alerts (30, 15, 7 days before expiry)
*Priority: P2 (Medium)*

#### **3.1.3 AI Assistant Requirements**

**AI-FR-001: Zoe - Executive Assistant**
Zoe shall provide executive oversight with:
1. **Executive Summary Tab**: Real-time KPI dashboard (8 key metrics)
2. **AI Suggestions Tab**: Aggregated recommendations from all other assistants
3. **Calendar Tab**: Integrated schedule with Google Calendar sync
4. **Team Overview Tab**: Agent performance and availability
5. **Reports Tab**: One-click report generation (PDF/Excel)
*Priority: P1 (High)*

**AI-FR-002: Aurora - CTO Dashboard**
Aurora shall provide system monitoring with:
1. **System Health Tab**: Live service status (MongoDB, APIs, External services)
2. **API Monitor Tab**: Endpoint performance and error rates
3. **Documentation Tab**: Searchable technical documentation
4. **Deployments Tab**: Release history and rollback controls
*Priority: P1 (High)*

**AI-FR-003: Linda - WhatsApp CRM Manager**
Linda shall manage WhatsApp communications with:
1. **Active Conversations Tab**: Real-time chat interface
2. **Agent Assignment Tab**: Manual and automatic agent assignment
3. **Templates Tab**: Pre-approved message templates
4. **Analytics Tab**: Conversation metrics and response times
*Priority: P1 (High)*

**AI-FR-004: Clara - Lead Pipeline Manager**
Clara shall manage the sales pipeline with:
1. **Pipeline View Tab**: Kanban-style lead progression board
2. **Lead Qualification Tab**: Scoring and categorization tools
3. **Conversion Tracking Tab**: Lead-to-deal conversion analytics
4. **Automation Rules Tab**: Workflow automation configuration
*Priority: P1 (High)*

**AI-FR-005: Agent Assignment Engine**
The system shall automatically assign leads to agents using weighted scoring:
1. **Expertise Match** (35%): Agent specialization vs. property type/location
2. **Availability** (25%): Current workload and capacity
3. **Performance** (20%): Historical conversion rates and client ratings
4. **Proximity** (10%): Geographic closeness to property
5. **Client Match** (10%): Language, nationality, budget alignment
*Priority: P1 (High)*

**AI-FR-006: Chatbot Service**
The system shall provide AI-powered chat with:
1. **Bilingual Support**: English and Arabic language understanding
2. **Intent Recognition**: 10+ predefined real estate intents
3. **Context Management**: Remember conversation history within session
4. **Lead Qualification**: Automatic scoring based on conversation content
5. **Agent Handoff**: Seamless transfer to human agent when needed
*Priority: P2 (Medium)*

#### **3.1.4 Property Management Requirements**

**PM-FR-001: Property Listing Management**
The system shall manage property listings with:
1. **UAE-Specific Fields**: RERA number, DLD permit, DEWA number, Ejari status
2. **Media Management**: High-resolution images, virtual tours, floor plans
3. **Multi-Portal Publishing**: One-click publishing to Bayut, Property Finder, Dubizzle
4. **Status Tracking**: Draft → Active → Under Offer → Sold/Rented workflow
*Priority: P1 (High)*

**PM-FR-002: Advanced Property Search**
The system shall provide property search with:
1. **Location Filtering**: By emirate, community, and neighborhood
2. **Price Range**: Minimum and maximum in AED
3. **Property Type**: Villa, apartment, townhouse, penthouse, land
4. **Amenities**: Swimming pool, gym, concierge, parking, etc.
5. **Advanced Filters**: Year built, square footage, bedroom/bathroom count
*Priority: P2 (Medium)*

**PM-FR-003: DAMAC Hills 2 Inventory Integration**
The system shall integrate with DAMAC inventory database:
1. **Automatic Synchronization**: Daily sync of available units
2. **Price Monitoring**: Track price changes and trends
3. **Availability Alerts**: Notify when specific unit types become available
4. **Competitive Analysis**: Compare with similar properties in area
*Priority: P2 (Medium)*

#### **3.1.5 Integration Requirements**

**INT-FR-001: WhatsApp Business API Integration**
The system shall integrate with WhatsApp Business API to:
1. **Receive Messages**: Process incoming customer messages via webhook
2. **Send Messages**: Send templated and free-form messages
3. **Template Management**: Create and approve message templates
4. **Session Management**: Maintain conversation context across messages
5. **Media Handling**: Send and receive images, documents, location
*Priority: P1 (High)*

**INT-FR-002: Google Workspace Integration**
The system shall integrate with Google services:
1. **Google Calendar**: Create viewing appointments and sync with agent calendars
2. **Google Drive**: Store and manage documents (contracts, images, reports)
3. **Google Contacts**: Sync client contact information
4. **Single Sign-On**: Optional Google-based authentication
*Priority: P2 (Medium)*

**INT-FR-003: Payment Gateway Integration**
The system shall integrate with payment processors:
1. **Stripe**: Primary payment processing for commissions and fees
2. **Mashreq NEOBiz**: Local UAE bank integration for wire transfers
3. **Aani**: QR code payment generation for quick transactions
4. **Receipt Generation**: Automatic digital receipt creation
*Priority: P3 (Low)*

### **3.2 Non-Functional Requirements**

#### **3.2.1 Performance Requirements**
1. **Page Load Time**: Dashboard initial load < 3 seconds on 50 Mbps connection
2. **API Response Time**: 95% of API calls respond in < 500ms under normal load
3. **Concurrent Users**: Support 50 simultaneous users without degradation
4. **Data Refresh**: Real-time data updates within 5 seconds of backend change
5. **Search Performance**: Property search results in < 2 seconds with 10,000+ listings

#### **3.2.2 Security Requirements**
1. **Authentication**: JWT-based authentication with 24-hour expiry
2. **Authorization**: Role-based access control (RBAC) for all features
3. **Data Encryption**: AES-256 encryption for data at rest and TLS 1.3 for data in transit
4. **Input Validation**: All user inputs validated server-side before processing
5. **Audit Logging**: All sensitive operations logged with user ID and timestamp

#### **3.2.3 Usability Requirements**
1. **Learnability**: New users should be able to perform basic tasks within 15 minutes
2. **Efficiency**: Frequent tasks should be accomplishable within 3 clicks
3. **Error Recovery**: Users should recover from errors with clear guidance
4. **Accessibility**: WCAG 2.1 Level AA compliance for users with disabilities
5. **Responsive Design**: Full functionality on mobile, tablet, and desktop

#### **3.2.4 Reliability Requirements**
1. **Availability**: 99.5% uptime excluding scheduled maintenance
2. **Mean Time Between Failures (MTBF)**: > 720 hours (30 days)
3. **Mean Time To Recovery (MTTR)**: < 30 minutes for critical failures
4. **Data Integrity**: Automatic daily backups with 30-day retention
5. **Error Handling**: Graceful degradation when external services fail

#### **3.2.5 Scalability Requirements**
1. **Horizontal Scaling**: Ability to add application servers without downtime
2. **Database Scaling**: MongoDB sharding support for 1M+ documents
3. **Cache Strategy**: Redis caching for frequently accessed data
4. **Load Balancing**: Support for multiple concurrent instances
5. **Storage Growth**: Support for 100GB+ of document storage

### **3.3 External Interface Requirements**

#### **3.3.1 User Interfaces**
1. **Primary Web Interface**: Responsive React application (Chrome, Firefox, Safari, Edge)
2. **Admin Interface**: Enhanced controls for company owner and administrators
3. **Mobile-Optimized View**: Touch-friendly interface for phones and tablets
4. **Print-Optimized Views**: Contract and report printing formats

#### **3.3.2 Hardware Interfaces**
1. **Server Hardware**: Cloud-based (AWS/Azure/Google Cloud) virtual machines
2. **Client Hardware**: Modern browsers on desktops, tablets, and smartphones
3. **Peripheral Support**: Webcam for document scanning, printers for contracts

#### **3.3.3 Software Interfaces**
1. **MongoDB Database**: MongoDB Atlas cloud database
2. **Firebase Authentication**: User identity and authentication management
3. **WhatsApp Business API**: Cloud API for WhatsApp messaging
4. **Stripe API**: Payment processing and subscription management
5. **Google APIs**: Calendar, Drive, and Maps integration

#### **3.3.4 Communication Interfaces**
1. **HTTPS/REST**: JSON-based REST API for all client-server communication
2. **WebSocket**: Real-time notifications and dashboard updates
3. **Webhook**: Incoming webhooks for WhatsApp and payment notifications
4. **Email/SMS**: Transactional notifications to users
